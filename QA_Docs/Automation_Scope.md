# Automation Scope

## 1. 자동화 대상

본 프로젝트에서는 BAT 관점에서 빠르게 검증 가능한 항목을 자동화 대상으로 선정합니다.

본 포트폴리오에서 BAT는 상세 게임 플레이 기능 검증이 아니라,  
신규 빌드 또는 프로젝트 변경 후 기본 실행 가능 여부를 빠르게 확인하는 테스트로 정의합니다.

자동화 대상은 다음 기준을 따릅니다.

- 결과가 명확하게 Pass/Fail로 판단 가능한 항목
- 반복 실행이 필요한 항목
- 빌드 안정성 확인에 직접적으로 관련된 항목
- Unity Test Framework로 검증 가능한 항목
- 실패 시 빌드 검토가 필요한 항목

## 2. 자동화 가능 항목

| 항목 | 자동화 가능 여부 | 사유 |
|---|---|---|
| Unity 프로젝트 기본 설정 확인 | 가능 | Unity 버전 및 주요 설정 파일을 기준으로 확인 가능 |
| TitleScreen 씬 로드 확인 | 가능 | 씬 로드 성공 여부를 코드로 검증 가능 |
| SampleLevel 씬 로드 확인 | 가능 | 주요 플레이 씬이 오류 없이 로드되는지 확인 가능 |
| Play Mode 진입 확인 | 가능 | 테스트 실행 중 예외 발생 여부 확인 가능 |
| 필수 오브젝트 존재 확인 | 가능 | Camera, GameManager, TitleMenu 등 기본 오브젝트 존재 여부 검증 가능 |
| 주요 Staging 씬 로드 확인 | 가능 | 기능별 샘플 씬이 로드 가능한 상태인지 확인 가능 |
| 치명적 에러 로그 확인 | 부분 가능 | 테스트 실행 중 Error 또는 Exception 로그 발생 여부를 확인 가능 |

## 3. 수동 테스트 또는 별도 테스트로 분류한 항

| 항목 | 수동 분류 사유 |
|---|---|
| 이동/점프 세부 동작 검증 | 기능 테스트 범위에 가까움 |
| 적 충돌 및 대미지 검증 | 기능 테스트 또는 회귀 테스트 범위에 가까움 |
| 문/스위치 동작 검증 | 기능 테스트 범위에 가까움 |
| 조작감 | 주관적 판단이 필요함 |
| 난이도 | 플레이어 경험 기반 판단이 필요함 |
| 그래픽 품질 | 시각적 검수가 필요함 |
| 사운드 품질 | 청각적 검수가 필요함 |
| 레벨 디자인 만족도 | 정성적 판단이 필요함 |

## 4. 자동화 적용 계획

1. Unity 프로젝트 실행 환경 구성
2. 테스트 대상 씬 확인
3. BAT 기준 테스트 케이스 정리
4. Unity Test Framework 설치 또는 활성화 확인
5. Play Mode 테스트 폴더 생성
6. 씬 로드 테스트 작성
7. 필수 오브젝트 존재 확인 테스트 작성
8. 테스트 실행 중 치명적 에러 발생 여부 확인
9. GitHub Actions 연동
10. Discord 알림 연동
11. 테스트 결과 문서화

## 5. 한계

본 포트폴리오는 공개 Unity 프로젝트를 테스트 대상으로 활용한 QA 자동화 학습 및 포트폴리오 목적의 프로젝트입니다.

상용 게임 클라이언트의 전체 QA 프로세스를 대체하는 것이 아니라,  
BAT 자동화 설계 및 적용 가능성을 보여주는 것을 목적으로 합니다.

또한 본 포트폴리오의 BAT 자동화 범위는 빌드 기본 동작 검증에 초점을 두며,  
상세 게임 플레이 기능 검증, 조작감 검증, 밸런스 검증, 전체 회귀 테스트는 포함하지 않습니다.
