# Automation Scope

## 1. 자동화 대상

본 프로젝트에서는 BAT 관점에서 빠르게 검증 가능한 항목을 자동화 대상으로 선정합니다.

자동화 대상은 다음 기준을 따릅니다.

- 결과가 명확하게 Pass/Fail로 판단 가능한 항목
- 반복 실행이 필요한 항목
- 빌드 안정성 확인에 직접적으로 관련된 항목
- Unity Test Framework로 검증 가능한 항목

## 2. 자동화 가능 항목

| 항목 | 자동화 가능 여부 | 사유 |
|---|---|---|
| 씬 로드 확인 | 가능 | 씬 로드 성공 여부를 코드로 검증 가능 |
| Player 오브젝트 존재 확인 | 가능 | 태그 또는 오브젝트 이름으로 검색 가능 |
| Main Camera 존재 확인 | 가능 | 필수 오브젝트 존재 여부 검증 가능 |
| Play Mode 진입 확인 | 가능 | 테스트 실행 중 예외 발생 여부 확인 가능 |
| 이동/점프 기능 확인 | 부분 가능 | 위치, 속도, 상태 값 변화로 일부 검증 가능 |
| 상호작용 오브젝트 존재 확인 | 가능 | 필수 오브젝트 누락 여부 검증 가능 |

## 3. 수동 테스트로 분류한 항목

| 항목 | 수동 분류 사유 |
|---|---|
| 조작감 | 주관적 판단이 필요함 |
| 난이도 | 플레이어 경험 기반 판단이 필요함 |
| 그래픽 품질 | 시각적 검수가 필요함 |
| 사운드 품질 | 청각적 검수가 필요함 |
| 레벨 디자인 만족도 | 정성적 판단이 필요함 |

## 4. 자동화 적용 계획

1. Unity 프로젝트 실행 환경 구성
2. Unity Test Framework 설치 또는 활성화 확인
3. Play Mode 테스트 폴더 생성
4. 씬 로드 테스트 작성
5. Player 오브젝트 존재 확인 테스트 작성
6. GitHub Actions 연동
7. Discord 알림 연동
8. 테스트 결과 문서화

## 5. 한계

본 포트폴리오는 공개 Unity 프로젝트를 테스트 대상으로 활용한 QA 자동화 학습 및 포트폴리오 목적의 프로젝트입니다.

상용 게임 클라이언트의 전체 QA 프로세스를 대체하는 것이 아니라,  
BAT 자동화 설계 및 적용 가능성을 보여주는 것을 목적으로 합니다.
