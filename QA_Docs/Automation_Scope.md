# Automation Scope
 
## 1. 자동화 대상
 
본 프로젝트에서는 BAT 및 기능 테스트 관점에서 자동화 가능한 항목을 선정합니다.
 
자동화 대상은 다음 기준을 따릅니다.
 
- 결과가 명확하게 Pass/Fail로 판단 가능한 항목
- 반복 실행이 필요한 항목
- 빌드 안정성 또는 기능 동작 확인에 직접적으로 관련된 항목
- Unity Test Framework로 검증 가능한 항목
---
 
## 2. 자동화 구현 항목
 
### BAT 테스트 (BATSceneLoadTests.cs)
 
| 항목 | 자동화 여부 | 사유 |
|---|---|---|
| TitleScreen 씬 로드 확인 | ✅ 완료 | 씬 로드 성공 여부를 코드로 검증 가능 |
| TitleScreen 필수 오브젝트 존재 확인 | ✅ 완료 | 태그 기반으로 오브젝트 존재 여부 검증 가능 |
| SampleLevel 씬 로드 확인 | ✅ 완료 | 씬 로드 성공 여부를 코드로 검증 가능 |
 
### 기능 테스트 — 플레이어 이동 (PlayerMovementTests.cs)
 
| 항목 | 자동화 여부 | 사유 |
|---|---|---|
| 플레이어 오브젝트 존재 확인 | ✅ 완료 | 태그 기반으로 검증 가능 |
| 좌우 이동 확인 | ✅ 완료 | Move() 호출 후 포지션 변화로 검증 가능 |
| 이동 중 IsMoving 상태 확인 | ✅ 완료 | public 프로퍼티로 직접 검증 가능 |
| 점프 확인 | ✅ 완료 | Jump() 호출 후 Rigidbody2D velocity로 검증 가능 |
| 점프 중 IsJumping 상태 확인 | ✅ 완료 | public 프로퍼티로 직접 검증 가능 |
| 이중 점프 방지 확인 | ✅ 완료 | 공중 재점프 시도 후 velocity 변화로 검증 가능 |
 
### 기능 테스트 — 전투/공격 (PlayerCombatTests.cs)
 
| 항목 | 자동화 여부 | 사유 |
|---|---|---|
| Actor/Thrower 컴포넌트 존재 확인 | ✅ 완료 | GetComponent로 검증 가능 |
| 초기 HP 확인 | ✅ 완료 | public 변수로 직접 검증 가능 |
| 투사체 생성 확인 | ✅ 완료 | ThrowProjectile() 호출 후 씬 오브젝트 수 변화로 검증 가능 |
| 피격 시 HP 감소 확인 | ✅ 완료 | Damage 태그 콜라이더 생성으로 피격 시뮬레이션 가능 |
| HP 0 시 사망 처리 확인 | ✅ 완료 | Health 직접 설정 후 피격으로 검증 가능 |
| 무적시간 적용 확인 | ✅ 완료 | 피격 후 HP 변화 없음으로 검증 가능 |
 
### 기능 테스트 — UI/메뉴 (UIMenuTests.cs)
 
| 항목 | 자동화 여부 | 사유 |
|---|---|---|
| StartGameText 존재 및 활성화 확인 | ✅ 완료 | public 변수로 직접 검증 가능 |
| 게임 시작 시 StartGameText 비활성화 확인 | ✅ 완료 | StartGame() 코루틴 실행 후 검증 가능 |
| 게임 시작 중복 실행 방지 확인 | ✅ 완료 | PendingStartGame 프로퍼티로 검증 가능 |
| 오디오 할당 확인 | ✅ 완료 | public 변수 null 여부로 검증 가능 |
| 배경음악 실제 재생 여부 확인 | ❌ 제외 | CI 배치모드에서 오디오 재생 불가 |
 
### 기능 테스트 — 씬 전환 (SceneTransitionTests.cs)
 
| 항목 | 자동화 여부 | 사유 |
|---|---|---|
| TitleScreen → SampleLevel 전환 확인 | ✅ 완료 | StartGame() 실행 후 씬 이름으로 검증 가능 |
| SampleLevel buildIndex 확인 | ✅ 완료 | buildIndex 값으로 직접 검증 가능 |
| 씬 전환 후 Player 존재 확인 | ✅ 완료 | 태그 기반으로 검증 가능 |

## 3. 수동 테스트로 분류한 항목
 
| 항목 | 수동 분류 사유 |
|---|---|
| 조작감 | 주관적 판단이 필요함 |
| 난이도 | 플레이어 경험 기반 판단이 필요함 |
| 그래픽 품질 | 시각적 검수가 필요함 |
| 사운드 품질 | 청각적 검수가 필요함 (CI 배치모드에서 재생 불가) |
| 레벨 디자인 만족도 | 정성적 판단이 필요함 |
| 플랫폼별 호환성 | 별도 환경 구성이 필요함 |
 
---
 
## 4. 한계
 
본 포트폴리오는 공개 Unity 프로젝트를 테스트 대상으로 활용한 QA 자동화 학습 및 포트폴리오 목적의 프로젝트입니다.
 
상용 게임 클라이언트의 전체 QA 프로세스를 대체하는 것이 아니라,  
BAT 및 기능 테스트 자동화 설계 및 적용 가능성을 보여주는 것을 목적으로 합니다.
 
CI 배치모드 환경의 특성상 오디오 재생 여부는 자동화 검증이 불가하며,  
이 경우 할당 여부(Inspector 연결)로 대체 검증했습니다.
