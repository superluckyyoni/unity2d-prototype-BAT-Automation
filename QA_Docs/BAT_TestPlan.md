# BAT Test Plan

## 1. 목적

본 문서는 공개 Unity 2D 플랫포머 프로젝트를 대상으로,  
빌드 수락 테스트(BAT, Build Acceptance Test) 범위를 정의하고  
자동화 테스트 적용 계획을 정리하기 위해 작성되었습니다.

본 포트폴리오는 전체 회귀 테스트가 아닌,  
신규 빌드가 기본적으로 실행 가능한 상태인지 확인하는 핵심 기능 위주의 BAT 자동화를 목표로 합니다.

## 2. 테스트 대상

- Project: Prototype - 2D Platformer Game
- Original Repository: https://github.com/practical-works/unity2d-prototype
- Unity Version: 2022.1.10f1
- License: MIT License
- Test Framework: Unity Test Framework

## 3. BAT 범위

본 프로젝트에서는 다음과 같은 핵심 기능을 BAT 대상으로 선정합니다.

- 프로젝트가 정상적으로 열리는지 확인
- 메인 씬이 오류 없이 로드되는지 확인
- 플레이어 오브젝트가 정상적으로 존재하는지 확인
- Play Mode 진입 시 예외가 발생하지 않는지 확인
- 기본 이동 또는 점프 기능이 동작하는지 확인
- 주요 상호작용 오브젝트가 누락되지 않았는지 확인

## 4. 자동화 전략

- Unity Test Framework를 사용하여 자동화 테스트를 작성합니다.
- 씬 로드, 오브젝트 존재 여부, Play Mode 진입 확인은 Play Mode 테스트로 분류합니다.
- 단순 데이터 또는 로직 검증이 가능한 항목은 Edit Mode 테스트로 분류합니다.
- 시각적 품질, 조작감, 난이도 판단 등 주관적 검증 항목은 수동 테스트 대상으로 분류합니다.

## 5. 제외 범위

다음 항목은 본 포트폴리오의 자동화 범위에서 제외합니다.

- 전체 회귀 테스트
- 그래픽/아트 품질 검수
- 사운드 품질 검수
- 조작감, 타격감 등 주관적 품질 평가
- 플랫폼별 호환성 테스트
- 실제 상용 빌드 배포 검증

## 6. 기대 효과

본 BAT 자동화를 통해 신규 변경 사항 반영 후  
기본 실행 가능 여부와 핵심 오브젝트 누락 여부를 빠르게 확인할 수 있습니다.

또한 GitHub Actions 및 테스트 결과 알림과 연동하여,  
빌드 검증 과정의 반복 작업을 줄이는 것을 목표로 합니다.
